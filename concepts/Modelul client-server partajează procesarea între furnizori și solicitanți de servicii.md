# Modelul client-server partajează procesarea între furnizori și solicitanți de servicii

Serverul este o aplicație (proces) care oferă servicii utilizatorilor, iar clientul este o aplicație care solicită servicii. Clientul inițiază comunicația, serverul răspunde. Un server poate oferi mai multe servicii simultan (DNS, HTTP, SMTP, FTP, SSH). Legătura dintre client și server se numește sesiune. [[Nivelurile superioare OSI asigură interfața cu utilizatorul și gestionarea dialogurilor]] implementează acest model prin protocoale de nivel aplicație.
