# Protocolul OPC permite interoperabilitatea între aplicații și echipamente de automatizare

OPC rezolvă problemele driverelor independente: conflicte între drivere, incompatibilități la schimbări hardware, acces concurent la dispozitive. Specificațiile OPC includ: Data Access (acces date în timp real), Alarms & Events (alarme și evenimente), Batch (procese batch), Historical Data Access (date istorice), Security (securitate), XML (cross-platform) și UA (Unified Architecture, independent de Windows). OPC XML și OPC UA elimină limitarea la sistemul de operare Windows.
