# Ars Contexta Methodology Reference

Portable, domain-agnostic distillation of Tools for Thought for Agents research.

## Core Philosophy

**If it won't exist next session, write it down now.**
**Depth over breadth. Quality over speed.**
**The vault is the argument.**

## Why This Works: Cognitive Science Foundations

- **Working Memory Limits (Cowan's 4±1)** — Agents need external structures
- **Extended Mind (Clark & Chalmers)** — Vault participates in cognition
- **Spreading Activation** — Wiki-links implement multi-hop reasoning
- **Small-World Topology** — Dense clusters + long-range bridges
- **Context-Switching Cost** — 23-minute reorientation penalty

## Universal Note Pattern

1. **Prose-as-Title** — Title IS the insight, works as proposition when linked
2. **YAML Frontmatter** — description, type, created
3. **Body** — Reasoning with connective words
4. **Wiki-Links** — [[note title]] creates graph edges
5. **Topics Footer** — Links to at least one MOC

## Universal Operations

- Capture — Zero friction entry
- Process — Transform raw to structured
- Connect — Link related knowledge
- Navigate — Manage attention via MOCs
- Maintain — Health checks, orphan detection
- Synthesize — Cross-topic patterns
- Retrieve — Progressive disclosure
- Evolve — Challenge assumptions, improve system

## MOC Architecture

Three-tier: Hub -> Domain -> Topic -> Notes

## Session Rhythm

Orient -> Work -> Persist

## Self-Extension Principle

After scaffolding, the agent should build its own hooks, skills, schema, and connectors.

## Key Research Claims

- Wiki links implement GraphRAG without infrastructure
- Each new note compounds value by creating traversal paths
- Descriptions are retrieval filters not summaries
- Fresh context per task preserves quality
- Throughput matters more than accumulation
- Backward maintenance asks what would be different if written today
