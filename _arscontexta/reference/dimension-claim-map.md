# Dimension-Claim Map

Maps research claims to configuration dimensions.

## Granularity
- Atomic needs explicit linking (else orphan drift)
- Atomic + heavy processing = coherent (processing sustains atomic demands)
- Atomic + light processing = unsustainable
- Claims: composability test, prose-as-title, one claim per note

## Organization
- Flat-associative adapts; hierarchy brittles
- Flat + 2-tier nav + volume > 50 = navigation vertigo
- MOCs add hierarchy without folder nesting
- Claims: flat-associative over hierarchical for agent systems

## Linking
- Dense connections are the primary value mechanism
- Explicit + implicit needs semantic search compensation
- Claims: wiki links = graph edges, each new link compounds value

## Processing
- Heavy processing at capture time ensures quality
- Light processing needs strong retrieval mechanisms
- Processing discipline lapses → maintenance debt
- Claims: throughput > accumulation, temporal separation

## Navigation
- 3-4 tier manages attention at scale
- 2-tier works for volume < 100
- Claims: MOC hierarchy for attention management

## Maintenance
- Condition-based replaces time-based
- Tight for high-change domains, lax for stable
- Maintenance debt accumulates silently
- Claims: backward maintenance, condition-based triggers

## Schema
- Moderate enables querying without impeding capture
- Dense needs validation support (template + automation)
- Minimal works for low-processing systems
- Claims: description over schema, evolve fields from query needs

## Automation
- Convention is the safe starting point (Gall's Law)
- Full automation needs platform support
- Hooks verify; skills judge (determinism boundary)
- Claims: convention trending toward automation at friction points
