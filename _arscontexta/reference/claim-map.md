# Research Claim Map

Core claims organized by topic area. Foundation for derivation reasoning.

## Graph Structure
- Wiki links implement GraphRAG without infrastructure
- Each new note compounds value by creating traversal paths
- Flat-associative organization adapts while hierarchy brittles
- Dense connections are the primary value mechanism

## Agent Cognition
- Session handoff creates continuity without persistent memory
- LLM attention degrades as context fills
- Fresh context per task preserves quality better than chaining
- Instructions early in context followed more reliably than late

## Discovery / Retrieval
- Descriptions are retrieval filters, not summaries
- Discovery-first design optimizes for future agent discovery
- Semantic search enables cross-vocabulary discovery
- Progressive disclosure: description before content

## Processing Workflow
- Throughput matters more than accumulation
- Processing at capture time ensures quality
- Temporal separation of capture/processing preserves freshness
- WIP limits force processing over accumulation

## Note Design
- Prose-as-title works as a proposition when linked
- Markdown files + YAML frontmatter + wiki links = graph DB
- Description (~150 chars) adds information beyond the title
- Topics footer provides MOC membership

## Maintenance / Evolution
- Backward maintenance asks "what would be different if written today?"
- Maintenance debt accumulates when processing discipline lapses
- Condition-based triggers replace time-based scheduling
- Never auto-implement identity changes
