Definitii, Istoric si Caracteristici PHP
Contextul Web 풉칂칬 Client si Server
PHP nu poate fi inteles fara sa stii cum functioneaza un site web. Exista doua entitati care comunica in permanenta:
CLIENT (browser) SERVER (Apache / Nginx)
Chrome, Firefox, Safari, Edge Ruleaza PHP, Python, Node.js etc.
Interpreteaza HTML, CSS, JavaScript Acceseaza baze de date (MySQL)
Ruleaza pe calculatorul utilizatorului Citeste / scrie fisiere pe disc
Trimite cereri HTTP (REQUEST) Primeste cereri HTTP (REQUEST)
Primeste si afiseaza HTML (RESPONSE) Executa PHP si trimite HTML
NU stie nimic despre PHP (RESPONSE)
Browserul NU vede niciodata codul PHP
Fluxul complet al unei cereri web 풉칂칬 pas cu pas:
a) Utilizatorul tasteaza www.exemplu.ro in browser si apasa Enter
b) Browserul face o cerere HTTP GET catre serverul www.exemplu.ro
c) Serverul primeste cererea si identifica fisierul index.php
d) Serverul porneste interpretorul PHP care executa scriptul
e) PHP acceseaza baza de date, citeste fisiere, calculeaza 풉칂칬 produce HTML pur
f) Serverul trimite HTML-ul catre browser (browserul NU primeste cod PHP)
g) Browserul interpreteaza HTML-ul si afiseaza pagina utilizatorului
PHP = limbaj EXCLUSIV server-side. Codul PHP niciodata nu ajunge la browser.
Browserul primeste mereu HTML pur, indiferent ce limbaj a generat acel HTML.
Tocmai de aceea PHP poate accesa baze de date si fisiere fara riscuri de securitate directe.
PHP (acronim recursiv: PHP: Hypertext Preprocessor) este un limbaj de programare de tip scripting, general-purpose cu
accent pe web, open-source, creat special pentru generarea dinamica de pagini HTML pe server.
PHP este un limbaj de scripting interpretat, server-side, cu tipizare dinamica, incorporabil direct in HTML, proiectat
pentru dezvoltarea rapida a aplicatiilor web.
Codul PHP este executat pe server; rezultatul (de obicei HTML) este transmis clientului.
Versiunea curenta stabila: PHP 8.3 (2023)
Licenta: PHP License (compatibila GPL, open-source)

$schimbat = true; } }
if (!$schimbat) break; // deja sortat }
return $arr;}
